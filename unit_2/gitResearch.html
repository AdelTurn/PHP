A. Version Control Software

A version control software tracks changes to files like additions and deletions. This is important because if multiple people are working on a project a version control software can track who made what changes, can resolve conflicts in code, and undo changes to code easily.

B. Add

Adds files to the stage before the local repo, in preperation to be commited.

C. commit

Commits the files added by "add" to the local repo, before being "pushed" into the remote repo.

D. Push

Pushes files from the local repo the the remote repo like github or bitbucket.

E. Pull

Pulls files from remote repo to your local machine.

F. Clone

Takes an existing repo and clones/creates a copy of it at a new directory.
